Google OAuth credentials need to be configured before users can log in.

This is a one-time setup that should be done by the system administrator.

1Create Google Cloud Project

  1. Go to the Google Cloud Console
  2. Create a new project or select an existing one
  3. Name it "GN Ticket Automator" or similar

2Enable APIs

  1. In your Google Cloud project, go to "APIs & Services" â†’ "Library"
  2. Search for and enable "Google+ API"
  3. Also enable "Google OAuth2 API" if available

3Create OAuth Credentials

  1. Go to "APIs & Services" â†’ "Credentials"
  2. Click "Create Credentials" â†’ "OAuth client ID"
  3. Choose "Web application"
  4. Name it "GN Ticket Automator"
  5. Add authorized redirect URIs:
    http://localhost:5000/oauth/callback
    https://yourdomain.com/oauth/callback
  6. Click "Create" and copy the Client ID and Client Secret

4Configure OAuth Consent Screen

  1. Go to "APIs & Services" â†’ "OAuth consent screen"
  2. Choose "Internal" for organization use only
  3. Fill in required information:
    • App name: GN Ticket Automator
    • User support email: Admin email
    • Authorized domains: takingitglobal.org
  4. Add scopes: email, profile, openid

5Update Application Code

Edit the main.py file and replace these lines:

# Find these lines in main.py:
GOOGLE_CLIENT_ID = "YOUR_GOOGLE_CLIENT_ID_HERE"
GOOGLE_CLIENT_SECRET = "YOUR_GOOGLE_CLIENT_SECRET_HERE"

# Replace with your actual credentials:
GOOGLE_CLIENT_ID = "your_actual_client_id_from_google_cloud"
GOOGLE_CLIENT_SECRET = "your_actual_client_secret_from_google_cloud"

Security Note: Keep these credentials secure and never commit them to version control.

6Update Allowed Domains (Optional)

If you want to allow users from domains other than takingitglobal.org, edit this line in main.py:

# Current setting:
ALLOWED_DOMAINS = ["takingitglobal.org"]

# To allow multiple domains:
ALLOWED_DOMAINS = ["takingitglobal.org", "yourdomain.com"]

7Test the Setup

  1. Save the main.py file with your OAuth credentials
  2. Restart the application
  3. Visit the homepage - Google login should now work
  4. Test with a user from your allowed domain
